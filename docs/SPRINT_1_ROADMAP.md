# Sprint 1 Roadmap

## Sprint Goal

Turn the WUCHENG pilot outline into a production-ready teaser package with script direction, first storyboard coverage, visual exploration targets, and asset tracking.

## Deliverables

| Workstream | Deliverable | Target State |
| --- | --- | --- |
| Script | Pilot Script V1 | Drafted from `script/pilot/pilot_outline_v1.md`. |
| Storyboard | Sequence 001-005 board plan | Shot list or board placeholders for the teaser. |
| Concept Art | Gray Harbor visual references | Draft references for street, alley, dream box, and blue dream. |
| Character | Qi Zhou visual direction | First-pass silhouette, wardrobe, and posture notes. |
| Assets | Asset Index V1 | All draft production assets tracked in `assets/ASSET_INDEX.md`. |
| Production | Handbook V1 | Working rules captured in `production/PRODUCTION_HANDBOOK.md`. |

## Work Plan

### 1. Script Package

- Convert pilot outline beats into a timed script draft.
- Keep dialogue minimal and avoid exposition.
- Mark sequence IDs so storyboard and assets can reference them.

### 2. Storyboard Package

- Create one storyboard plan per sequence.
- Track shot purpose, timing, and required assets.
- Preserve the rule: one shot, one piece of information.

### 3. Concept Art Package

- Define visual boards for Gray Harbor, Qi Zhou, the dream box, and the blue dream.
- Keep blue absent from normal-world references.
- Record prompts, sources, and review notes in the asset index.

### 4. Asset System

- Assign asset IDs before review.
- Keep draft, review, approved, and retired states visible.
- Avoid using unindexed assets in production packages.

### 5. Production Review

- Review script, storyboard, concept, and asset index together.
- Capture open questions in the handbook or next sprint notes.
- Identify the minimum package needed for a 90-second pilot teaser review.

## Acceptance Criteria

- `README.md` explains the workspace and current focus.
- `CODEX.md` defines project engineering rules.
- `production/PRODUCTION_HANDBOOK.md` defines production skeleton and review gates.
- `assets/ASSET_INDEX.md` contains the initial register and intake template.
- `docs/SPRINT_1_ROADMAP.md` defines Sprint 1 goals, deliverables, and acceptance criteria.
- New storyboard and concept art folders have clear starting points.

## Risks

- Visual exploration may drift into neon or spectacle too early.
- AI outputs may lose provenance if prompt and rights notes are not captured immediately.
- The pilot can become over-explained if script work ignores the outline's restraint.

## Definition of Done

Sprint 1 is complete when the team can review a coherent teaser package: script draft, storyboard plan, concept targets, indexed assets, and open questions.
