# WUCHENG Production Handbook

## Purpose

This handbook defines the working rules for producing WUCHENG across script, storyboard, concept art, generated assets, and review cycles.

## Project Identity

- Title: WUCHENG
- Format: AI film project
- Current target: 90-second pilot teaser, then EP01 expansion
- Core setting: Gray Harbor
- Current protagonist: Qi Zhou
- Creative mode: quiet mystery, tactile realism, restrained impossible image

## Creative Pillars

- Ordinary first, impossible second.
- Respectful observation before spectacle.
- One shot, one piece of information.
- No cyberpunk neon.
- The blue dream is an exception, not a palette.

## Directory Protocol

| Directory | Rule |
| --- | --- |
| `script/` | Keep outlines, scripts, beat sheets, and dialogue drafts here. |
| `storyboard/` | Store storyboard frames, shot lists, timing notes, and animatic exports. |
| `concept_art/` | Store visual explorations by character, setting, prop, and palette. |
| `assets/` | Track all generated or sourced production assets through the asset index. |
| `production/` | Store production rules, review checklists, delivery specs, and handbook updates. |

## Asset Lifecycle

| Status | Meaning |
| --- | --- |
| `proposed` | Idea or placeholder exists, not yet reviewed. |
| `draft` | Asset exists and can be discussed. |
| `review` | Ready for creative or technical review. |
| `approved` | Accepted for use in the current production package. |
| `retired` | Preserved for history, no longer in active use. |

## Scene Package Standard

Each scene or sequence package should include:

- sequence ID and title
- script excerpt or beat summary
- storyboard frames or shot list
- concept references
- asset IDs
- timing notes
- open questions

## AI Generation Protocol

Record the following for generated assets:

- tool or model
- prompt
- negative prompt, if used
- seed or version settings, if available
- source references
- output path
- edit history
- rights or licensing note

## Review Gates

| Gate | Required Evidence |
| --- | --- |
| Script review | Script or outline path, revision number, owner, open questions. |
| Visual review | Asset IDs, storyboard frames, concept references, palette notes. |
| Production review | Delivery format, naming, asset index state, unresolved risks. |
